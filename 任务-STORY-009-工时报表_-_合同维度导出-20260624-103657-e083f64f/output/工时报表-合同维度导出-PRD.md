# 工时报表 - 合同维度导出方案

## 1. 方案定位

本方案用于 `STORY-003` 的技术验证产物，目标是在工时报表中提供按合同聚合、筛选、预览和导出的能力，支持项目管理、成本核算、供应商结算和合同履约分析。

当前输入目录仅包含任务标题和作业上下文，未提供真实业务制度、字段字典、权限矩阵或现有页面截图。因此以下内容以可落地的产品方案表达，所有涉及生产系统现状、字段来源、审批口径、金额展示和导出阈值的内容均标记为【待确认】或【待研发确认】。

## 2. 业务目标

1. 在工时报表中新增“合同维度”统计口径，帮助业务人员快速查看某一合同周期内发生的工时量、人员投入和结算相关数据。
2. 支持按合同、项目、供应商、时间区间、审批状态等条件筛选工时记录，并导出标准 Excel 文件。
3. 对缺少合同归属的工时数据进行显性提示，避免导出结果被误用于结算。
4. 保留导出任务、导出人、筛选条件和文件摘要，满足后续审计追踪。

## 3. 适用角色

| 角色 | 使用诉求 | 权限边界 |
| --- | --- | --- |
| 项目经理 | 查看项目下各合同工时投入情况 | 仅查看本人数据权限范围内项目和合同 |
| 合同/成本人员 | 核对合同维度工时，为结算和成本归集提供依据 | 金额字段是否可见【待确认】 |
| 供应商协同人员 | 查看本单位合同相关工时明细 | 仅限所属供应商或被授权合同【待确认】 |
| 平台管理员 | 配置导出权限、查看导出审计 | 不直接改变业务数据 |

## 4. 范围

### 4.1 本期纳入

- 工时报表新增合同维度筛选和导出能力。
- 支持“汇总表”和“明细表”两种导出口径。
- 支持按合同编号、合同名称、项目、供应商、工时日期、审批状态筛选。
- 导出文件中保留筛选条件、生成时间、导出人和数据口径说明。
- 对合同归属缺失、合同已失效、跨合同拆分等异常数据给出提示。

### 4.2 本期不纳入

- 不新增合同台账维护能力。
- 不新增工时审批流程。
- 不修改合同金额、结算单价、供应商主数据。
- 不自动生成结算单或付款申请。
- 不处理离线导入后的历史数据清洗，历史数据修复策略【待确认】。

## 5. 核心页面与交互

### 5.1 报表入口

在现有工时报表页面增加“统计维度”切换项：

- 人员维度【现有口径，待确认】
- 项目维度【现有口径，待确认】
- 合同维度

切换到“合同维度”后，列表默认展示合同汇总结果。筛选区固定在列表上方，导出按钮位于筛选区右侧或列表工具栏右侧，具体位置按现有前端规范复用【待确认】。

### 5.2 筛选条件

| 条件 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 工时日期 | 日期范围 | 本月 | 以工时发生日期为准，不以填报或审批日期为准 |
| 项目 | 下拉/树选 | 全部有权限项目 | 支持按项目过滤合同 |
| 合同 | 远程搜索下拉 | 全部有权限合同 | 支持合同编号、合同名称模糊搜索 |
| 供应商/乙方单位 | 下拉 | 全部 | 用于供应商结算核对【待确认】 |
| 工时状态 | 多选 | 已审批 | 默认仅导出已审批工时，是否允许导出草稿/驳回【待确认】 |
| 工时类型 | 多选 | 全部 | 正常工时、加班工时、调整工时等枚举【待确认】 |
| 导出口径 | 单选 | 汇总表 | 汇总表 / 明细表 |

合同维度汇总列表建议字段：

| 字段 | 说明 |

### 5.4 导出设置

点击“导出”后弹出导出设置弹窗：

- 导出口径：汇总表 / 明细表。
- 文件格式：`.xlsx`，大数据量是否支持 `.csv`【待研发确认】。
- 是否包含金额字段：仅有金额权限时可选【待确认】。
- 是否包含异常数据页：默认包含。
- 导出说明：自动写入文件首页或首行说明。

导出为异步任务时，按钮状态改为“生成中”，完成后提供下载入口；失败时展示失败原因和重试入口。

建议默认导出异常数据页，字段包括：

- 工时记录 ID
- 工时日期
- 人员
- 项目
- 原始合同字段
- 异常类型
- 异常说明
- 建议处理方式

异常类型示例：

- `missing_contract`：工时记录无合同归属。
- `inactive_contract`：工时日期不在合同有效期内【待确认】。
- `multi_contract_without_split`：同一工时记录关联多个合同但缺少拆分比例。
- `permission_filtered`：因数据权限被过滤，仅在审计记录中保留，不在用户文件中暴露明细【待确认】。

## 7. 数据归属与统计规则

### 7.1 合同归属优先级

建议按以下优先级识别工时所属合同：

1. 工时记录直接引用的合同 ID。
2. 工时关联任务、派工单、作业单上的合同 ID。
3. 项目和供应商绑定的默认合同。
4. 人工补充映射表或历史修复结果【待确认】。

若仍无法识别合同，则进入“未归属合同”分组，并在异常数据页展示。

### 7.2 时间口径

- 统计周期以工时发生日期为准。
- 审批状态筛选以导出时刻的最新状态为准。
- 跨月审批但工时发生在本月的记录，应计入本月工时【待确认】。

### 7.3 状态口径

默认仅纳入已审批通过的工时记录。是否允许选择“待审批、已驳回、草稿、已撤回”等状态需业务确认。

### 7.4 去重口径

- 工时记录以 `timesheet_id` 或系统唯一记录 ID 去重【待研发确认】。
- 人数以人员主键去重，不以姓名去重。
- 调整记录不得覆盖原始记录，应按冲正/补录规则独立体现【待确认】。

### 7.5 金额口径

金额字段属于高风险字段，建议默认不导出。仅当用户具备金额查看权限时展示以下字段：

- 单价
- 可结算工时
- 预估金额
- 币种/税率【待确认】

金额计算规则、单价来源、税前税后口径和合同限额校验均需业务与研发共同确认。

## 8. 权限与审计

### 8.1 权限控制

- 页面查看权限：有工时报表访问权限。
- 合同数据范围：按项目、组织、供应商或合同授权过滤【待确认】。
- 导出权限：单独控制，避免仅能查看但不能导出。
- 金额字段权限：单独控制，默认不可见。

### 8.2 导出审计

每次导出记录以下信息：

- 导出任务 ID
- 导出人、所属组织、角色
- 导出时间
- 筛选条件
- 导出口径
- 文件格式
- 导出行数
- 是否包含金额字段
- 文件 SHA256 或摘要【待研发确认】
- 成功/失败状态与失败原因

## 9. 异步任务与性能

建议前端点击导出后统一创建导出任务：

1. 前端提交筛选条件和导出口径。
2. 后端校验权限和参数。
3. 后端生成导出任务。
4. 小数据量可同步返回文件，大数据量进入异步队列【待研发确认】。
5. 用户在导出中心或当前页面下载文件。

建议阈值：

- 小于 5,000 行：可同步导出【待研发确认】。
- 5,000 至 100,000 行：异步导出【待研发确认】。
- 超过 100,000 行：提示缩小筛选范围或导出 CSV【待研发确认】。

## 11. 验收标准

1. 用户进入工时报表后，可以切换到合同维度。
2. 合同维度列表能按筛选条件展示合同汇总数据。
3. 默认只统计已审批工时，若业务确认允许其他状态，应按筛选项准确统计。
4. 同一人员在同一合同下多条工时记录，人数只计一次，工时按记录累加。
5. 同一工时记录不会在同一导出文件中重复计入。
6. 缺少合同归属的数据不会静默混入正常合同，必须进入异常提示或未归属分组。
7. 用户无金额权限时，页面和导出文件都不出现金额字段。
8. 用户无合同或项目数据权限时，无法查询或导出相关数据。
9. 导出文件包含汇总表或明细表，字段顺序与确认后的模板一致。
10. 导出文件包含筛选条件、导出时间、导出人和数据口径说明。
11. 大数据量导出不会阻塞页面主流程，导出失败时有明确失败原因。
12. 每次导出都有审计记录，可追溯导出人、筛选条件、行数和文件摘要。

## 13. 风险与待确认

- 【待确认】现有工时数据是否已经具备稳定的合同 ID。
- 【待确认】合同维度默认纳入哪些审批状态。
- 【待确认】供应商、班组、岗位等字段来源和脱敏规则。
- 【待确认】金额字段是否属于本期范围，以及金额权限控制规则。
- 【待确认】历史工时缺合同归属时的修复流程。
- 【待研发确认】导出行数阈值、异步任务队列、文件存储时效和下载鉴权。
- 【待研发确认】导出文件是否需要加水印、加密或下载链接有效期。
